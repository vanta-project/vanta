           Vanta Reliability and Control-Plane Specification
                       draft-vanta-reliability-00

Abstract

   This document defines the Vanta control plane, acknowledgement model,
   flow control, retransmission signaling, reconnection semantics, and
   effectively-once mutation behavior.

1.  Control Plane and Data Plane

   Vanta distinguishes control-plane traffic from data-plane traffic.
   Control-plane frames include acknowledgements, flow control, errors,
   heartbeats, stream management, and audit receipts.

   Control-plane traffic SHOULD be prioritized over application data
   traffic.

2.  Acknowledgement Model

   Vanta distinguishes transport acknowledgement from semantic
   acknowledgement.

   *  Transport acknowledgement confirms receipt and parse success.

   *  Semantic acknowledgement confirms application disposition.

3.  ACK Frame

   ACK frames confirm receipt of one or more frames on a target stream.
   They include at least a target stream identifier, the largest
   contiguous received sequence number, and optional selective
   acknowledgement ranges.

4.  NACK Frame

   NACK frames indicate missing or rejected frame sequences.  A NACK
   does not by itself require retransmission, but it provides an
   explicit loss or rejection signal.

5.  Flow Control

   Flow control is receiver-driven.  Receivers grant byte credits and
   optionally message credits.  Senders MUST stop sending data-plane
   traffic when credits are exhausted.

   Implementations SHOULD reserve capacity for control-plane traffic to
   avoid deadlock.

6.  Heartbeat

   HEARTBEAT frames are used for liveness detection and optional round-
   trip time measurement.

7.  Retransmission

   Retransmission is explicit.  Retransmitted frames MUST preserve
   MessageID and any relevant OperationID.  Retransmitted frames SHOULD
   set the RETRANSMIT flag.

8.  Effectively-Once Mutation Semantics

   Vanta targets effectively-once semantics for mutation requests.
   Duplicate detection is based on sender identity and mutation
   identifiers rather than transport guarantees alone.

   Receivers SHOULD maintain an in-memory deduplication cache for the
   active session lifetime.

9.  Mutation Dispositions

   *  APPLIED

   *  DUPLICATE_APPLIED

   *  DUPLICATE_REJECTED

   *  VALIDATION_FAILED

   *  CAPABILITY_DENIED

10.  Reconnection and Failure Semantics

   Reconnection uses policy-defined backoff and optional resumption
   support.  Unresolved mutations MUST be treated as
   UNKNOWN_AFTER_DISCONNECT until explicitly reconciled.

11.  Ordering

   Ordering may be negotiated or requested per stream.  The default
   SHOULD favor throughput and low latency unless stronger ordering
   semantics are explicitly requested.

12.  Error Handling

   ERROR frames provide explicit and structured protocol failures,
   including authentication errors, schema violations, flow-control
   failures, and routing or capability errors.
